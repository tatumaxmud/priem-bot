# 🚀 Размещение бота на PythonAnywhere (24/7)

Пошаговая инструкция, как сделать так, чтобы бот работал круглосуточно на
PythonAnywhere.

---

## Важно: как работает бесплатный тариф

PythonAnywhere — удобно, но у **бесплатного** тарифа есть два ограничения:

1. **Нельзя держать постоянно работающий процесс** (нет «Always-on task»).
   Поэтому обычный запуск `python bot.py` (опрос/polling) на бесплатном тарифе
   24/7 НЕ работает. Бесплатно бот работает через **webhook** — Telegram сам
   присылает сообщения на ваш сайт (для этого есть файл `flask_app.py`).

2. **Белый список сайтов для интернета.** Бесплатный аккаунт может обращаться
   только к разрешённым адресам. `api.telegram.org` — разрешён (бот работает),
   а `api.groq.com` и другие ИИ-сервисы — нет. Значит, **умные ИИ-ответы на
   бесплатном тарифе недоступны**. Бот будет отвечать через меню и поиск по базе
   знаний — это работает хорошо и закрывает большинство вопросов абитуриентов.

Итого — два варианта:

| | Бесплатный (Free) | Платный (Hacker, ~$5/мес) |
|---|---|---|
| Как работает | webhook + Flask (`flask_app.py`) | always-on task + polling (`bot.py`) |
| Умные ИИ-ответы | ❌ нет (белый список) | ✅ да (полный интернет) |
| Меню и поиск по базе | ✅ да | ✅ да |
| Админ-команды | ✅ да | ✅ да |
| Сложность настройки | средняя | простая |

Ниже — оба варианта. Начните с бесплатного.

## ВАРИАНТ А. Бесплатно (webhook)

### Шаг 1. Регистрация
Зарегистрируйтесь на https://www.pythonanywhere.com (тариф **Beginner — Free**).

### Шаг 2. Загрузить файлы бота
Проще всего через **Bash-консоль** (меню Consoles → Bash). Если проект в GitHub:
```bash
git clone https://github.com/ВАШ_АККАУНТ/telegram-bot.git
```
Либо вручную: меню **Files** → создайте папку `telegram-bot` и загрузите туда
`bot.py`, `flask_app.py`, `set_webhook.py`, `knowledge.md`, `requirements.txt`.

### Шаг 3. Установить библиотеки
В Bash-консоли:
```bash
pip install --user pyTelegramBotAPI Flask python-dotenv
```

### Шаг 4. Создать веб-приложение
1. Меню **Web** → **Add a new web app** → **Manual configuration** →
   выберите версию Python (например, 3.10).
2. Откроется страница настроек веб-приложения.

### Шаг 5. Прописать WSGI-файл
На странице **Web** найдите ссылку на **WSGI configuration file** и откройте её.
Удалите всё содержимое и вставьте (замените `ВАШ_ЛОГИН`):
```python
import os
import sys

path = "/home/ВАШ_ЛОГИН/telegram-bot"
if path not in sys.path:
    sys.path.insert(0, path)

# Переменные окружения
os.environ["BOT_TOKEN"] = "ТОКЕН_ОТ_BOTFATHER"
os.environ["ADMIN_IDS"] = "ВАШ_TELEGRAM_ID"
os.environ["WEBHOOK_SECRET"] = "pridumayte-svoy-sekret-123"
os.environ["KNOWLEDGE_FILE"] = "/home/ВАШ_ЛОГИН/telegram-bot/knowledge.md"
os.environ["DB_FILE"] = "/home/ВАШ_ЛОГИН/telegram-bot/bot.db"
# AI_API_KEY на бесплатном тарифе НЕ указываем (всё равно недоступен)

from flask_app import application
```
Сохраните файл.

### Шаг 6. Перезапустить веб-приложение
На странице **Web** нажмите зелёную кнопку **Reload**. Проверьте, открыв в браузере
`https://ВАШ_ЛОГИН.pythonanywhere.com` — должно появиться «Бот приёмной комиссии работает.».

### Шаг 7. Зарегистрировать webhook
В Bash-консоли (один раз):
```bash
cd ~/telegram-bot
export BOT_TOKEN="ТОКЕН_ОТ_BOTFATHER"
export WEBHOOK_SECRET="pridumayte-svoy-sekret-123"
export PA_USERNAME="ВАШ_ЛОГИН"
python set_webhook.py
```
Должно вывести «Webhook установлен на: https://ВАШ_ЛОГИН.pythonanywhere.com/pridumayte-svoy-sekret-123».

### Шаг 8. Проверить
Напишите боту `/start` в Telegram — появится меню. Готово, бот работает 24/7.

### Шаг 9. Продлевать раз в 3 месяца
Бесплатные веб-приложения нужно **продлевать раз в 3 месяца**: на странице **Web**
периодически появляется кнопка вида «Run until 3 months from today» — нажмите её,
чтобы бот не отключился.

### Как менять ответы бота
Отредактируйте `knowledge.md` (через меню **Files**) и отправьте боту команду
`/reload`, либо нажмите **Reload** на странице Web.

## ВАРИАНТ Б. Платный тариф Hacker (с ИИ, проще)

Если нужны умные ИИ-ответы, оформите тариф **Hacker (~$5/мес)** — он снимает белый
список и даёт «Always-on task». Тогда webhook не нужен, бот работает обычным опросом:

1. Загрузите файлы и установите библиотеки (шаги 2–3 выше; добавьте `openai`:
   `pip install --user openai`).
2. Создайте файл `.env` в папке бота (по образцу `.env.example`), укажите
   `BOT_TOKEN`, `ADMIN_IDS` и `AI_API_KEY` (ключ Groq).
3. Меню **Tasks** → **Always-on tasks** → добавьте команду:
   ```
   python3 /home/ВАШ_ЛОГИН/telegram-bot/bot.py
   ```
4. Если раньше включали webhook — удалите его:
   ```bash
   python set_webhook.py delete
   ```
5. Готово: бот работает 24/7 с ИИ-ответами.

## Частые проблемы
- **Бот не отвечает (Free).** Проверьте: веб-приложение Reload’нуто; webhook
  зарегистрирован (`python set_webhook.py`); `WEBHOOK_SECRET` одинаковый в WSGI-файле
  и при регистрации; в логах (**Web → Error log**) нет ошибок.
- **Открывается сайт, но бот молчит.** Не зарегистрирован webhook или не совпадает
  секрет в адресе.
- **Нет умных ответов.** На бесплатном тарифе это ожидаемо — ИИ заблокирован белым
  списком. Используйте меню/поиск или перейдите на тариф Hacker.
- **Бот выключился через время.** Не продлили бесплатное веб-приложение (Шаг 9).

> ⚠️ Не публикуйте токен бота в открытом доступе (например, в публичном GitHub).
> Храните его в WSGI-файле или `.env`, а `.env` держите в `.gitignore`.

> Условия бесплатных тарифов PythonAnywhere и ИИ-сервисов со временем меняются —
> перед запуском стоит свериться с актуальной документацией PythonAnywhere.
