# 🚀 Размещение бота на Render (бесплатно, 24/7, с ИИ)

Здесь бот работает через **polling** — значит доступны **умные ИИ-ответы**
(на Render нет белого списка, как у PythonAnywhere). Это рекомендуемый бесплатный
вариант.

> Я подготовил все нужные файлы (`run_render.py`, `render.yaml`, `Procfile`,
> `runtime.txt`). Вам остаётся залить проект в GitHub и сделать несколько кликов —
> создать аккаунт и привязать репозиторий за вас я не могу.

---

## Коротко про бесплатные тарифы (важно)

- **Render Free** — бесплатно, но веб-сервис **засыпает после ~15 минут** без
  обращений. Решение: внешний «будильник» (UptimeRobot) пингует сервис каждые
  5–10 минут — и бот не засыпает. Лимит бесплатного тарифа ~750 часов/мес
  (хватает на один постоянно работающий сервис).
- **Railway** — постоянного бесплатного тарифа больше нет: даётся разовый кредит
  $5 на 30 дней, затем нужен платный план (~$5/мес). Подходит, если готовы платить;
  инструкция в конце.

## ЧАСТЬ 1. Залить проект в GitHub (один раз)

1. Создайте аккаунт на https://github.com и новый репозиторий, например `priem-bot`
   (Private — приватный, чтобы токен не был виден).
2. Загрузите туда все файлы из папки `telegram-bot`:
   `bot.py`, `run_render.py`, `knowledge.md`, `requirements.txt`, `render.yaml`,
   `Procfile`, `runtime.txt`, `.gitignore`.
   - Через сайт: на странице репозитория **Add file → Upload files** → перетащите
     файлы → **Commit**.
   - Или через программу **GitHub Desktop**.
3. ⚠️ Файл `.env` НЕ загружайте (в нём токен). Он уже в `.gitignore`. Токен зададим
   на Render через переменные окружения.

## ЧАСТЬ 2. Развернуть на Render (бесплатно)

### Способ 1 — через Blueprint (проще, использует render.yaml)
1. Зарегистрируйтесь на https://render.com (можно войти через GitHub).
2. Нажмите **New +** → **Blueprint**.
3. Подключите свой репозиторий `priem-bot`. Render прочитает `render.yaml` и
   предложит создать сервис `priem-bot`.
4. Нажмите **Apply**. Сервис создастся на бесплатном плане.

### Способ 2 — вручную (если Blueprint не используете)
1. **New +** → **Web Service** → подключите репозиторий.
2. Настройки:
   - **Runtime:** Python 3
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python run_render.py`
   - **Instance Type:** **Free**

### Задать переменные окружения
В сервисе откройте **Environment** и добавьте:
| Ключ | Значение |
|------|----------|
| `BOT_TOKEN` | токен от @BotFather |
| `ADMIN_IDS` | ваш Telegram ID (узнать у @userinfobot) |
| `AI_API_KEY` | ключ Groq (https://console.groq.com) — для ИИ-ответов |
| `AI_BASE_URL` | `https://api.groq.com/openai/v1` |
| `AI_MODEL` | `llama-3.3-70b-versatile` |

Нажмите **Save**, дождитесь, пока сервис соберётся и появится статус **Live**.

### Проверить
Откройте адрес сервиса (вида `https://priem-bot.onrender.com`) — увидите
«Бот приёмной комиссии работает (polling)». Напишите боту `/start` в Telegram.

## ЧАСТЬ 3. Чтобы бот не засыпал (обязательно для Free)

1. Скопируйте адрес сервиса, добавьте `/health`, например:
   `https://priem-bot.onrender.com/health`
2. Зарегистрируйтесь на https://uptimerobot.com (бесплатно).
3. **Add New Monitor** → тип **HTTP(s)** → вставьте адрес `/health` →
   интервал **5 минут** → создать.

Теперь UptimeRobot будет «будить» сервис, и бот работает 24/7.

> Альтернатива UptimeRobot — https://cron-job.org (тоже бесплатно).

## Как менять ответы бота
1. Отредактируйте `knowledge.md` в репозитории GitHub (Edit → Commit).
2. Render автоматически пересоберёт сервис с новой базой.
   (Либо отправьте боту команду `/reload`, если правили файл прямо на сервере.)

## ЧАСТЬ 4. Railway (если готовы платить ~$5/мес)

1. Зарегистрируйтесь на https://railway.com → **New Project** →
   **Deploy from GitHub repo** → выберите `priem-bot`.
2. Railway сам определит Python и установит зависимости.
3. В **Variables** добавьте: `BOT_TOKEN`, `ADMIN_IDS`, `AI_API_KEY`, `AI_BASE_URL`,
   `AI_MODEL`.
4. Команда запуска (если попросит): `python bot.py` — на Railway бот может работать
   обычным polling без keep-alive (порт не нужен).
5. Учтите: после пробного периода Railway списывает оплату по тарифу Hobby (~$5/мес).

## Частые проблемы
- **Бот молчит / «409 Conflict» в логах.** Где-то параллельно запущен второй
  экземпляр того же бота (например, локально или старый webhook). Остановите
  лишние экземпляры; при необходимости выполните удаление webhook:
  `python set_webhook.py delete`.
- **Сервис «спит», отвечает с задержкой.** Не настроен пинг (Часть 3) или интервал
  слишком большой — поставьте 5 минут.
- **Нет ИИ-ответов.** Не задан `AI_API_KEY` или исчерпан лимит Groq — бот ответит
  поиском по базе.
- **Сборка падает.** Проверьте, что в репозитории есть `requirements.txt` и
  `run_render.py`.

> Условия бесплатных тарифов Render, Railway, UptimeRobot и ИИ-сервисов со временем
> меняются — перед запуском сверьтесь с актуальной документацией.
